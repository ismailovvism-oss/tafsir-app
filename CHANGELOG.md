## 2026-06-25 — Колонки: исправлено налезание строк (grid → таблица)

### Исправлено
- В режиме «Колонки» строки **налезали друг на друга** (и на телефоне, и на
  десктопе): при гибких колонках CSS-grid занижал высоту auto-ряда (считал текст
  «в одну строку», без учёта переноса) → ряд ~44px, контент переполнял ячейку.
  Переписано с grid на обычную **`<table>`** (`table-layout:fixed`): ряд `tr`
  растёт по самой высокой ячейке, перенос корректный, колонки равны и выровнены,
  горизонтальный скролл через `min-width` таблицы, липкая шапка `thead`.
  Сворачивание длинных тафсиров и сохранение позиции при тоггле сохранены.

## 2026-06-25 — Колонки: выравнивание аятов между колонками (CSS-grid)

### Исправлено
- В режиме «Колонки» аяты **не совпадали по горизонтали** между колонками
  (и с арабским оригиналом): колонки были независимыми flex-стеками, высота
  ячейки зависела от длины текста → ряды разъезжались. Переписано на **CSS-grid
  с построчным выводом** (ряд = аят, высота ряда = по самой высокой ячейке) —
  теперь один аят стоит в один ряд во всех колонках. Горизонтальный скролл и
  липкая шапка сохранены; селекторы `.col-layout/.col-col` → `.col-grid`.

### Добавлено

  `mawsua_masur`, полный свод тафсира **бил-ма'сур** (предания саляфов, все 114

  из [spa5k/tafsir_api](https://github.com/spa5k/tafsir_api) (зеркало quran.com),
  сборка скриптом `build_mawsua_masur.py`.
- **Источник целиком вынесен на Cloudflare R2** (~120 МБ: монолит + чанки +
  индекс) — в репозиторий НЕ коммитится (см. `.gitignore`). Это первый
  «тяжёлый» текстовый источник на внешнем хранилище.

### Механизм (новое)
- **Переопределение хранилища на уровне источника**: в `config` у источника
  поле **`dataBase`** (база URL). `resolveUrl` применяет её для по-аятных классов
  (`tafsir_text`, `tafsir_monolith`, `search_index`, `quran_text`); остальные
  (лёгкие) классы остаются локальными. Раньше базу можно было менять только на
  весь КЛАСС сразу — теперь и для одного источника. Позволяет держать лёгкие
  источники в репо, а тяжёлые — на R2, не меняя вызывающий код.

## 2026-06 — Тафсир Ибн Касир на русском (новый источник)

### Добавлено
- **Источник «Тафсир Ибн Касир (рус.)»** (`kathir_ru`, группа «Тафсиры на
  русском») — русский **Мухтасар** (сокращённый) Ибн Касира, перевод Айрата
  Вахитова. Покрытие: **суры 1–19 и 59–114** (73 суры, ~3000 аятов, fill ~50%).
  Помечен `caution` + тег `draft` (частичный, не проверен).

### Данные
- `build_kathir_ru.py` — пайплайн из TXT (pdftotext оригиналов) → per-аят JSON
  `data/tafsirs/kathir_ru/<сура>.json` + сводный `kathir_ru.json` + `index.json`.
  Исходные TXT — вне репо (Obsidian-вольт пользователя); промежуточные `.md` —
  в `data/_sources/kathir_ru/` (gitignored, пересоздаются скриптом).
- **Две версии PDF-экстракции**: A (центрированная проза, суры 1–8; арабский
  битый — вырезан) и B (двухколоночная вёрстка с номерами аятов, 11–19/59–114;
  арабский чистый, с огласовками).
- **Арабский**: формат B — канонический Uthmani из `_arabic.json`, подставлен по
  номеру аята (1942 аята). Формат A — битый скелет вырезан.
- **Сегментация по аятам**: маркеры `(N)` (формат A) и `**N.**` (формат B).
- **Сноски** формата A → `[^N]` + определения `[^N]:` под аятом (извлечены из
  «подвалов» страниц, перестали рвать предложения).
- **Диапазоны**: Мухтасар комментирует группами аятов → блок под первым аятом
  («*Аяты N–M*»), аятам-участникам — отсылка на первый аят.
- Пересборка: `python3 build_kathir_ru.py && python3 compute_fill.py && python3 sync_config.py`.

### Добавлено

### Исправлено

  аятах) — добавлен `flex:1 + overflow-y:auto`, как у режима «Аяты».
- **Мусхаф: единый кегль строк.** Раньше каждая строка масштабировалась
  отдельно (короткие «раздувались» до 1.5×) — теперь вся страница масштабируется
  одним коэффициентом по самой широкой строке: все глифы одного размера,
  короткие строки просто уже и центрируются. Постраничность не тронута.

### Добавлено ранее
- **Режим «Слова» (пословный перевод).** Третий режим чтения рядом с
  «Аяты» и «Мусхаф»: сетка слов аята, под каждым словом — английский глосс.
  Каждое слово — кликабельный элемент с адресом `data-loc="сура:аят:слово"`.
- **Грамматическая карточка из корпуса.** По тапу на слово (в режиме «Слова»
  и в режиме «Аяты») карточка строится **на лету по адресу слова** из
  морфологии: сегменты слова (приставки/основа/окончание), часть речи
  по-русски с цветом по части речи, у основы — корень и лемма, род/число/падеж,
  для глагола — время/залог/порода. Карточка больше **не носит текст в себе** —
  тянет данные по адресу. Старый ручной синтаксис `[[видимое::текст]]` сохранён.

### Данные
- `build_morph.py` собирает два слоя, связанных адресом слова `сура:аят:слово`,
  **порезанных по сурам** (как QPC-данные — ленивая загрузка нужной суры):
  - `data/wbw/words/<s>.json` — пословный глосс `{ "en", "ru_ref"? }` (77 429 слов);
  - `data/wbw/morph/<s>.json` — морфология: массив сегментов на слово (77 429 слов).
  Источники: морфология — [Quranic Arabic Corpus](http://corpus.quran.com)
  (арабская графика, репозиторий `mustafa0x/quran-morphology`, GPL); глоссы —
  `hablullah/data-quran` (CC BY-NC-ND, выровнены 1:1 с корпусом, проверено).
  Атрибуция в `data/wbw/ATTRIBUTION.txt` и в самом приложении.
- **Английский глосс** показывается дословно (display-only, без изменений).
  **Русский глосс** из data-quran — слабого качества (перевод через английский),
  хранится только как **скрытое сверочное** поле `ru_ref`, пользователю не
  показывается. Наш будущий русский перевод (`ru`) — отдельный пайплайн (Шаг 4),
  здесь не создаётся; `ru` и `ru_ref` не смешиваются.

  - Кнопка «↺ По умолчанию».
- **Навигация (нижняя панель)** в обоих режимах: единица

  страницы. Заголовок показывает «Сура · Джуз N · стр. P/604».

- **Карточки по нажатию** `[[слово::текст]]` — кликабельное слово открывает

- **Сворачивание длинных тафсиров** (>600 символов) с кнопкой «Развернуть».
