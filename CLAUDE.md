## Что это
**Библиотека тафсиров** — статический веб-сайт для чтения Корана, переводов и
тафсиров с телефона и десктопа. Без сборки и фреймворков: один `index.html`
(ванильный JS) + данные в `data/` (JSON). Хостится как статика (GitHub Pages).

## Как запустить / проверить
- Локально: `python3 -m http.server 8000` в корне → http://localhost:8000/
  (открывать `file://` нельзя — браузер блокирует загрузку JSON; для offline есть
  `.js`-зеркала, см. ниже).
- Тестов нет; проверка — глазами в браузере. Питон-скрипты идемпотентны.

## Стек и структура
```
index.html          ← всё приложение (UI + рендер + поиск), ванильный JS
data/
  config.json       ← ЕДИНЫЙ ИСТОЧНИК ПРАВДЫ: список источников и групп
  config.js         ← offline-зеркало (генерится sync_config.py — НЕ править руками)
  tafsirs/
    <id>.json       ← весь источник {сура:{аят:текст}} (для поиска)
    <id>/<surah>.json ← чанк по суре {аят:текст} (ленивая загрузка loadChunk)
    <id>/index.json ← список заполненных сур
    _arabic.json / _arabic_clean.json / _arabic_rasm.json ← канонический Коран
  index/<id>.json   ← инвертированный индекс поиска (build_index.py)
  qpc/v1/           ← данные печатного мусхафа (build_qpc.py)
  wbw/              ← пословный перевод + морфология (build_morph.py)
  about/<id>.md     ← вступление «о переводе» (кнопка ℹ); about/app.md = экран «О программе»
  img/<id>/         ← скан-источники kind:"image": КАРТЫ страниц (<sura>.json,
                      _about.json) ЛЕЖАТ ЗДЕСЬ; сами webp-страницы — на R2 (gitignored)
  _sources/         ← СЫРЬЁ сборок (gitignored! пересоздаётся скриптами)
fonts/              ← самохостимые шрифты (.woff2)
```

## Расположение данных (абстракция источников)
- **Реестр `SOURCES`** в начале `<script>` в `index.html` — для каждого КЛАССА
  ресурса `{baseUrl, pathTemplate}`. Классы: `config`, `coverage`, `quran_text`,
  `tafsir_text`, `tafsir_monolith`, `search_index`, `glosses`, `morphology`,
  `about`, `mushaf_glyphs`, `mushaf_fonts`, **`mushaf_scans`** (webp-страницы скана),
  **`scan_maps`** (карты-списки страниц скана), `audio` (задел), `arabic_volumes` (задел).
- **Все URL к данным — только через `resolveUrl(class, params)`.** Прямых/«зашитых»
  путей в коде нет. Чтобы перенести класс на другое хранилище, меняется ТОЛЬКО
  `baseUrl` нужного класса в `SOURCES` — вызывающий код не трогается; разные классы
  могут жить на разных базах.
- Текущие базы: почти всё `data`; `mushaf_fonts="fonts"`; **`mushaf_scans` →
  Cloudflare R2** (`https://pub-d99a56b61d914461b5864f421a013097.r2.dev`) — см.
  «Сканы на R2» ниже.

## Модель данных (главное)
- Каждый тафсир/перевод/словарь = `{ "<номер_аята>": "markdown-текст" }` по сурам.
- **`config.json` — единственное место правки** метаданных источника. После любой
  правки запусти **`python3 sync_config.py`** → пересоберёт `config.js`.
- Поле `fill` (доля заполненных аятов) **не править руками** —
  **`python3 compute_fill.py`** пересчитывает по данным.
- Поля источника: `id, file, name, nameAr, author, group, kind(quran|translation|
  tafsir|image), lang(ru|ar), era, tags[], year, fill`. Опционально: `about, caution,
  approved, translator, source…` (см. GUIDE §2).
- Группы (`config.json.groups`): quran / translations / russian / modern_ar / classical_ar.

## Разметка текста тафсиров (подмножество Markdown, рендер в index.html)
- `**жирный**`, `__жирный__`, `*курсив*`, `` `моно` ``, `# … ####`, `> цитата`,
  списки, таблицы, `---`.
- **ВАЖНО:** одиночное `_подчёркивание_` НЕ курсив (только `*звёздочки*`).
- Сноски: `текст[^1]` + отдельной строкой `[^1]: примечание` → список под аятом.
- Карточки: `[[видимое::текст]]` → кликабельное слово с попапом (грамматика).
- Текст HTML-экранируется (произвольный HTML вставить нельзя).

## Текущее состояние и подсистемы (что уже есть в коде)
Подробная история — в `CHANGELOG.md` («новое сверху»). Кратко, последнее сверху:
- **Сканы Богуславского — на Cloudflare R2** (см. след. раздел). webp убраны из
  репо и из истории git; карты страниц остались локально.
- **Экран «О программе»** — кнопка ℹ в шапке (в любом режиме). Текст —
  `data/about/app.md` (через класс `about` + `mdRender`), правится без кода;
  контакт-канал ссылкой. Markdown-ссылки `[текст](url)` рендерятся (новая вкладка).
- **Абстракция расположения данных** — реестр `SOURCES` + `resolveUrl` (см. выше).
- **Система ПОКРЫТИЯ** источников: индекс `data/coverage.json` (`build_coverage.py`);
  счётчик у аята «🌐 переводы · 📚 тафсиры»; попап со списком источников у аята
  («только здесь» / «постоянно»); **4-й режим «Источник»** (вкладка рядом с
  Аяты/Слова/Мусхаф) — экран выбора = компактная карта покрытия, чтение по покрытию,
  переключатель разрывов (серым/схлопнуть/скрыть), навигация по покрытым аятам.
  Скан-источники (kind:image) в покрытие НЕ входят (нет по-аятного покрытия).
- **4 режима чтения**: Аяты / Слова (пословный разбор + грам-карточка из корпуса) /
  Мусхаф (постраничный QPC, 604 стр.) / Источник. Плюс: заметки (инлайн +
  полноэкранный редактор; подключаются как слой-«источник» `_notes`), иерархические
  теги, закладки, поиск (инвертированный индекс), бесконтактный/иммерсивный режимы,
  скан-читалка, ⚙ Вид (шрифты/размеры/темы).

  Это источник истины имён страниц: **нумерация с пропусками** (напр. p015 нет) —
  приложение берёт имена ТОЛЬКО из карт, не предполагает сплошную нумерацию.
- Добавить новые страницы: залить webp на R2 в `img/<id>/`, обновить карты в
  `data/img/<id>/`. Менять хранилище — только `SOURCES.mushaf_scans.baseUrl`.

## Проверка headless (нет node/playwright, есть google-chrome-stable)
Сервер: `python3 -m http.server 8000`. Снять DOM/скриншот:
`google-chrome-stable --headless --no-sandbox --disable-gpu --hide-scrollbars
--window-size=440,950 --user-data-dir=<свежий> --virtual-time-budget=12000
--run-all-compositor-stages-before-draw --dump-dom|--screenshot=out.png URL`.
- Состояние через URL: `#сура` (хэш) и `?param` (query) — **query идёт ПЕРЕД хэшем**:
  `?x=1#103`, не `#103?x=1` (иначе попадёт во фрагмент).
- Для экранов, требующих действия (грам-карточка, попап, поиск, редактор, режим
  источника) — временно вставить авто-вызов в конце `loadConfig` (`setTimeout(...)`),
  снять, и ОТКАТИТЬ файл (md5/бэкап). Так делались скриншоты в `screenshots/`.
- Пара граблей: `window.innerWidth` врёт (clamp ~500) — адаптив проверять через
  `matchMedia`; динамический IntersectionObserver после JS-ре-рендера под
  virtual-time часто не срабатывает — для виртуализации грузить целевую суру
  естественной загрузкой; `let`-глобалы (ST, COV) не висят на `window`, а
  function-declaration (resolveUrl, doSearch) — доступны.
- JS-синтаксис без браузера: извлечь большой `<script>` и `node --check`.

## Рабочие правила
- Меняешь `config.json` → запусти `sync_config.py`. Меняешь данные источника →
  `compute_fill.py` И `build_coverage.py` (индекс покрытия для счётчика «Тафсиров: N»,
  списка у аята, режима источника и карты покрытия). Не редактируй `config.js`,
  `fill` и `coverage.json` вручную.
- `data/_sources/` в `.gitignore` — туда кладём сырьё/промежуточное, в коммит не идёт.
- Коммиты и changelog — по-русски, «новое сверху». Коммитить/пушить — только по
  просьбе пользователя.
- Для inline-обработчиков в HTML экранируй значения через `escAttr()`.
