# CLAUDE.md — ориентир для ассистента

Памятка для нового сеанса Claude. Подробности — в `GUIDE.md` (мейнтейнер),
`HELP.md` (пользователь), `CHANGELOG.md` (история), `BACKLOG.md` (планы).

---

## Что это
**Библиотека тафсиров** — статический веб-сайт для чтения Корана, переводов и
тафсиров с телефона и десктопа. Без сборки и фреймворков: один `index.html`
(ванильный JS) + данные в `data/` (JSON). Хостится как статика (GitHub Pages).

## Как запустить / проверить
- Локально: `python3 -m http.server 8000` в корне → http://localhost:8000/
  (открывать `file://` нельзя — браузер блокирует загрузку JSON; для offline есть
  `.js`-зеркала, см. ниже).
- Тестов нет; проверка — глазами в браузере. Питон-скрипты идемпотентны.

## Стек и структура
```
index.html          ← всё приложение (UI + рендер + поиск), ванильный JS
data/
  config.json       ← ЕДИНЫЙ ИСТОЧНИК ПРАВДЫ: список источников и групп
  config.js         ← offline-зеркало (генерится sync_config.py — НЕ править руками)
  tafsirs/
    <id>.json       ← весь источник {сура:{аят:текст}} (для поиска)
    <id>/<surah>.json ← чанк по суре {аят:текст} (ленивая загрузка loadChunk)
    <id>/index.json ← список заполненных сур
    _arabic.json / _arabic_clean.json / _arabic_rasm.json ← канонический Коран
  index/<id>.json   ← инвертированный индекс поиска (build_index.py)
  qpc/v1/           ← данные печатного мусхафа (build_qpc.py)
  wbw/              ← пословный перевод + морфология (build_morph.py)
  about/<id>.md     ← вступление «о переводе» (кнопка ℹ)
  img/<id>/         ← сканы для kind:"image"
  _sources/         ← СЫРЬЁ сборок (gitignored! пересоздаётся скриптами)
fonts/              ← самохостимые шрифты (.woff2)
```

## Модель данных (главное)
- Каждый тафсир/перевод/словарь = `{ "<номер_аята>": "markdown-текст" }` по сурам.
- **`config.json` — единственное место правки** метаданных источника. После любой
  правки запусти **`python3 sync_config.py`** → пересоберёт `config.js`.
- Поле `fill` (доля заполненных аятов) **не править руками** —
  **`python3 compute_fill.py`** пересчитывает по данным.
- Поля источника: `id, file, name, nameAr, author, group, kind(quran|translation|
  tafsir|image), lang(ru|ar), era, tags[], year, fill`. Опционально: `about, caution,
  approved, translator, source…` (см. GUIDE §2).
- Группы (`config.json.groups`): quran / translations / russian / modern_ar / classical_ar.

## Разметка текста тафсиров (подмножество Markdown, рендер в index.html)
- `**жирный**`, `__жирный__`, `*курсив*`, `` `моно` ``, `# … ####`, `> цитата`,
  списки, таблицы, `---`.
- **ВАЖНО:** одиночное `_подчёркивание_` НЕ курсив (только `*звёздочки*`).
- Сноски: `текст[^1]` + отдельной строкой `[^1]: примечание` → список под аятом.
- Карточки: `[[видимое::текст]]` → кликабельное слово с попапом (грамматика).
- Текст HTML-экранируется (произвольный HTML вставить нельзя).

## Скрипты сборки данных
| Скрипт | Назначение |
|---|---|
| `sync_config.py` | `config.json` → `config.js` (запускать после правки конфига) |
| `compute_fill.py` | пересчёт `fill` у всех источников |
| `build_coverage.py` | индекс покрытия `data/coverage.json` (какие аяты есть у источника) |
| `convert.py` | текстовый файл → чанки JSON + индекс (см. GUIDE §5) |
| `split.py` | монолит `tafsirs/<id>.json` → чанки по сурам |
| `build_index.py` | инвертированный индекс поиска `data/index/<id>.json` |
| `build_qpc.py` | глифы печатного мусхафа (QPC v1) |
| `build_morph.py` | пословный перевод + морфология (`data/wbw`) |
| `build_kathir_ru.py` | импорт русского Ибн Касира (см. ниже) |
| `validate_data.py` | проверка целостности данных |

## Рабочие правила
- Меняешь `config.json` → запусти `sync_config.py`. Меняешь данные источника →
  `compute_fill.py` И `build_coverage.py` (индекс покрытия для счётчика «Тафсиров: N»,
  списка у аята, режима источника и карты покрытия). Не редактируй `config.js`,
  `fill` и `coverage.json` вручную.
- `data/_sources/` в `.gitignore` — туда кладём сырьё/промежуточное, в коммит не идёт.
- Коммиты и changelog — по-русски, «новое сверху». Коммитить/пушить — только по
  просьбе пользователя.
- Для inline-обработчиков в HTML экранируй значения через `escAttr()`.
