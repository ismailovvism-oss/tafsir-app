# FACTSHEET — Библиотека тафсиров

Инвентаризация фактов о приложении: функции (как в коде), состав источников
(строго из `data/config.json` + `data/coverage.json`), техническая часть и
скриншоты. Нейтрально, без оценок. Собрано в режиме read-only, код не менялся.

Дата сбора: 2026-06-25.

---

## 1. ФУНКЦИИ

### Чтение
- **Режим «Аяты»** — поаятный показ: арабский оригинал + включённые переводы/тафсиры под каждым аятом.
- **Режим «Слова»** — пословный разбор: каждое арабское слово с русским подстрочником (глоссом); тап по слову открывает грамматическую карточку из корпуса (часть речи, падеж, корень, лемма, ссылка на Quranic Arabic Corpus).
- **Режим «Мусхаф»** — постраничный печатный мусхаф (мадани), страница за страницей, аутентичная вёрстка глифами (QPC v1), 604 страницы, маркеры концов аятов и заголовки сур.
- **Режим «Источник»** — чтение по покрытию одного выбранного перевода/тафсира (см. «Тафсиры и переводы»).
- **Несколько вариантов арабского текста**: с огласовками (усманский), без огласовок, без точек (расм) — переключаются в ⚙ Вид.
- **Поаятный показ ar/ru** — кнопка «ru»/«ar» у аята: когда включён только арабский, разово показать перевод этого аята (и наоборот).
- **Пять арабских начертаний**: Насх, Каллиграфический, Амири, Печатный (QPC, по умолчанию), Куфи.
- **Раскладка «Колонки/Список»** (на широком экране) — источники рядом или стопкой.
- **Сворачивание длинных толкований** — кнопка «Развернуть ▾» у объёмных записей.

### Навигация
- **Нижняя панель**: единица листания (Сура / Страница / Джуз / Хизб) + выпадающий выбор + стрелки ‹ ›.
- **Свайп** влево/вправо по области чтения — переход к предыдущему/следующему разделу.
- **Боковая панель**: список сур, сгруппированный по джузам (с переходом к началу джуза); вкладки «Суры», «🏷 Теги», «★ Закл.».
- **Deep-link** по URL-хэшу `#сура[:аят]`; восстановление **последней позиции** чтения автоматически (отдельно от закладок).
- **Введение к суре** (мукаддима/фадаиль) — блок над аятом 1, если есть в данных.

### Изучение языка
- **Пословный перевод** (глосс) по каждой суре.
- **Грамматическая карточка** на слово: морфология из корпуса (POS, падеж, число/род, корень, лемма).
- **Карточки в тексте** — синтаксис `[[видимое::текст]]` рендерит кликабельное слово с попапом-разбором.

### Тафсиры и переводы (система ПОКРЫТИЯ)
- **Счётчик у аята** «🌐 N · 📚 M» — сколько переводов и тафсиров есть у этого аята (по индексу покрытия).
- **Попап источников у аята** — список всех источников с пометкой есть/нет (✓/серый), сгруппирован «Переводы»/«Тафсиры» (сворачиваемые; переводы свёрнуты по умолчанию). Действия на источник: **«только здесь»** (разовый показ у этого аята) и **«постоянно»** (показывать всюду, где источник есть).
- **Режим источника** — чтение по покрытию одного источника; экран выбора = компактная карта покрытия (% и охват по каждому источнику). Непокрытые аяты: переключатель **серым / схлопнуть / скрыть**; навигация **«след./пред. покрытый аят»**; суры без источника приглушены в боковой панели и нижнем пикере.
- **Блочные тафсиры** — текст блока на первом аяте диапазона; на остальных аятах диапазона — отсылка «Разобрано в блоке X–Y → перейти».
- **Карта покрытия** — охват каждого источника (проценты, диапазоны сур), видно где «дыры».
- **Паспорт источника** (кнопка ℹ) — модалка «О переводе/тафсире»: тип, язык, эпоха, автор/переводчик, год, заполненность, файл + вступительный текст.
- **Индикаторы доверия/полноты** в каталоге: «✓ одобрено» / «⚠ не проверено» (caution) / «неполный» (draft) + полоска заполненности.
- **Сноски** под аятом — `текст[^N]` + определения, нумерованный список под аятом.
- **Markdown** в тексте источников: жирный/курсив/код/цитата/заголовки/списки/таблицы/`---`.

### Заметки и организация
- **Заметки к аяту** — инлайн-редактор под аятом + **полноэкранный редактор** с панелью форматирования (жирный, курсив, код, заголовок, цитата, списки, таблица, карточка) и вкладками «Правка»/«Просмотр».
- **Иерархические теги** — дерево тегов (создание/переименование/перемещение/удаление); тег на аят; родительский тег показывает аяты потомков; отдельная панель результатов по тегу.
- **Закладки** — ручные именованные, отдельная вкладка в сайдбаре.
- Заметки, теги, закладки, включённые слои — хранятся локально (localStorage).

### Особые режимы
- **Бесконтактный режим** (✋) — чтение и листание кнопочной панелью (‹ пред. / ▲ / ▼ / след. ›) без касания текста; в нём недоступен режим «Слова».
- **Иммерсивный режим** (⛶) — прячет всю обвязку, остаётся только поле чтения; использует Fullscreen API там, где доступно (с фолбэком).
- **Скан-читалка** (`kind:image`) — постраничный показ сканов для непарсируемых книг (вкладки «Текст»/«Примечания»); отдельный экран, взаимоисключающий с режимами чтения.

### Поиск
- **Полнотекстовый поиск** по текстам переводов/тафсиров (предсобранный инвертированный индекс на источник); подсветка совпадений, группировка по аятам, метка источника.
- Опция **«Искать во всех тафсирах»** (иначе — только во включённых).

### Настройки (⚙ Вид)
- Вариант(ы) арабского текста, арабское начертание.
- Отдельно размер шрифта арабского и русского; отдельно межстрочный интервал арабского и русского.
- Поля чтения, размер боковой панели, «Мусхаф на всю ширину», сброс к умолчаниям.
- **4 темы оформления**: зелёная, синяя, золотая, тёмная.

## 3. ТЕХНИЧЕСКОЕ

- **Тип**: статический одностраничный веб-сайт. Один файл `index.html` (ванильный JS, ~3,4 тыс. строк), без сборки и фреймворков. Данные — JSON в `data/`. Хостинг — GitHub Pages.
- **PWA**: **нет** (нет `manifest.json`, нет service worker). Обычный сайт; адаптив через CSS media-queries (`matchMedia`).
- **Данные** (`data/`):
  - `config.json` — реестр источников и групп (+ offline-зеркало `config.js`).
  - `coverage.json` — индекс покрытия: какие аяты есть у каждого источника (+ `coverage.js`).
  - `tafsirs/<id>/<сура>.json` — чанк по суре (ленивая загрузка, LRU-кэш); `tafsirs/<id>.json` — монолит (для поиска); `tafsirs/<id>/index.json` — список заполненных сур.
  - `index/<id>.json` — инвертированный индекс поиска по источнику.
  - `wbw/words/<сура>.json` — пословный глосс (en + русский подстрочник); `wbw/morph/<сура>.json` — морфология (грамматика). Полное покрытие 114 сур.
  - `qpc/v1/` — данные печатного мусхафа (мадани): `meta.json`, `page/<N>.json` (604 страницы), `surah/<сура>.json` (глифы-PUA по словам).
  - `img/<id>/` — сканы страниц (WebP) для `kind:image` + per-сура карта.
  - `about/<id>.md` — вступление/паспорт источника.
- **Корпус пословного разбора**: русский глосс + морфология на основе Quranic Arabic Corpus; лицензия **CC BY 4.0** (`dataset/`, `data/wbw/`, см. `ATTRIBUTION.txt`).
- **Шрифты** (`fonts/`, самохостинг `.woff2`): Reem Kufi, KFGQPC Uthmanic Hafs, Amiri Quran — арабские начертания; `qpc-v1/` — **604 постраничных глиф-шрифта** печатного мусхафа (по одному на страницу, инъекция `@font-face` лениво при показе).
- **Offline-режим**: опционально. Если присутствуют `window.TL_CONFIG` / `TL_DATA` / `TL_COVERAGE` (закомментированные `.js`-зеркала в `index.html`), они используются вместо `fetch` — работа без сервера. По умолчанию выключено.
- **Абстракция расположения данных**: реестр `SOURCES` + функция `resolveUrl(class, params)` — для каждого класса ресурса своя база (`baseUrl`) и шаблон пути; смена хранилища класса (напр. тяжёлый бинарь → объектное хранилище) — правкой одной настройки. Сейчас все базы указывают на текущее расположение.
- **Состояние пользователя** (localStorage): включённые слои, заметки, теги, закладки, настройки ⚙ Вид, тема, последняя позиция чтения.
- **Ленивая загрузка/кэш**: чанки по сурам (LRU), глосс/морфология по сурам, постраничные шрифты QPC по требованию; виртуализация тел аятов (IntersectionObserver).
- **Maintainer-скрипты** (Python, идемпотентные): `convert.py`, `split.py`, `build_index.py`, `compute_fill.py`, `build_coverage.py`, `build_qpc.py`, `build_morph.py`, `build_kathir_ru.py`, `sync_config.py`, `validate_data.py`, `fix_kathir_blocks.py`.
