# Как вручную создавать файлы тафсиров

## Рабочий процесс

### 1. Создайте файл

Откройте VS Code → File → New File → сохраните как `my_tafsir.json`
в папку `data/tafsirs/`

### 2. Начните с шаблона

Скопируйте этот шаблон и начните заполнять:

```json
{
  "1": {
    "1": "Тафсир к суре 1, аяту 1",
    "2": "Тафсир к суре 1, аяту 2"
  }
}
```

### 3. Добавляйте постепенно

Сегодня сделали суру 1 — сохранили, работает.
Завтра добавили суру 2 — сохранили, работает.
Нет никакой необходимости делать всё сразу.

### 4. Зарегистрируйте в config.json

Добавьте одну запись в `data/config.json` → массив `"tafsirs"`.
Это делается один раз, в самом начале.

## Пример: заполняем тафсир постепенно

### День 1 — только аль-Фатиха, частично:

```json
{
  "1": {
    "1": "Во имя Аллаха — начинаю с именем Аллаха...",
    "2": "Хвала — то есть полная и совершенная хвала принадлежит Аллаху..."
  }
}
```

Результат: аяты 1 и 2 суры 1 показываются. Остальные — "текст не найден".

### День 2 — дополнили Фатиху, начали Бакару:

```json
{
  "1": {
    "1": "Во имя Аллаха — начинаю с именем Аллаха...",
    "2": "Хвала — то есть полная и совершенная хвала принадлежит Аллаху...",
    "3": "Милостивому — Его милость объемлет все сущее...",
    "4": "Властелину Дня воздаяния — Он один владеет этим Днём...",
    "5": "Тебе одному поклоняемся — это суть единобожия...",
    "6": "Веди нас прямым путём — это мольба о руководстве...",
    "7": "Путём тех, кого Ты облагодетельствовал — пророков и праведников..."
  },
  "2": {
    "1": "Алиф Лям Мим — разрозненные буквы, о смысле которых Аллах знает лучше..."
  }
}
```

Результат: вся Фатиха работает. Из Бакары — только аят 1.

## Правила JSON, которые нужно знать

### Кавычки — только двойные "

✅ Правильно:
```json
"1": "текст тафсира"
```

❌ Неправильно:
```json
'1': 'текст тафсира'
```

### Запятые — после каждого элемента, КРОМЕ последнего

✅ Правильно:
```json
{
  "1": "первый аят",
  "2": "второй аят",
  "3": "третий аят"
}
```

❌ Неправильно (лишняя запятая после последнего):
```json
{
  "1": "первый аят",
  "2": "второй аят",
  "3": "третий аят",
}
```

### Кавычки внутри текста — экранируйте обратным слешем

Если в тексте тафсира есть кавычки, ставьте перед ними \

✅ Правильно:
```json
"1": "Сказал: \"Воистину, Аллах знает лучше\""
```

❌ Неправильно:
```json
"1": "Сказал: "Воистину, Аллах знает лучше""
```

### Переносы строк — используйте \n

JSON не поддерживает реальные переносы строк внутри значения.
Используйте \n там, где нужен новый абзац:

```json
"1": "Первый абзац тафсира.\n\nВторой абзац тафсира.\n\nТретий абзац."
```

## Подсказки в VS Code

1. **Установите расширение** — откройте Extensions (Ctrl+Shift+X),
   найдите "JSON" — VS Code уже имеет встроенную поддержку,
   но расширение "JSON Tools" добавляет форматирование

2. **Проверка ошибок** — VS Code подчёркивает ошибки красной волнистой линией.
   Если видите красное — значит где-то забыта кавычка, запятая или скобка

3. **Форматирование** — нажмите Shift+Alt+F (Windows) или Shift+Option+F (Mac)
   чтобы автоматически выровнять файл. Если файл не форматируется — в нём ошибка.

4. **Сворачивание** — кликните стрелку слева от номера строки,
   чтобы свернуть целую суру. Удобно при работе с большими файлами.

## Быстрый способ: вставка из текстового файла

Если у вас тафсир в виде простого текста, где каждый фрагмент
начинается с указания суры и аята, можно подготовить текстовый файл
в нашем формате и прогнать через convert.py:

```
### 1:1
Текст тафсира к суре 1, аяту 1.
Может быть на несколько абзацев.

### 1:2
Текст к суре 1, аяту 2.

### 2:1
Текст к суре 2, аяту 1.
```

Затем: `python3 convert.py my_tafsir.txt my_tafsir.json`

Этот способ проще, потому что не нужно думать о кавычках и запятых —
конвертер сделает всё сам.
